Project Title:
Standalone Modular RAG Platform for Learning and Reuse

Goal:
Build a reusable AI knowledge service that can be integrated into other projects. The main learning objective is to understand the full lifecycle of a RAG system, including ingestion, OCR, chunking, embeddings, vector search, retrieval, reranking, chat, voice, evaluation, and optional fine-tuning.

Primary Learning Desire:
I want this project to be designed as a step-by-step learning journey, not just a working product. For every phase, explain:
- what is being built
- why it is needed
- the concept behind it
- how it works internally
- trade-offs and alternatives
- how it connects to the next phase
- what I should learn before moving forward

Please keep the explanation beginner-friendly but also include practical architecture guidance, production considerations, and improvement ideas.

Project Vision:
A standalone AI service that can:
- ingest documents and structured data
- extract text from PDFs/images/scans using OCR
- chunk content intelligently
- generate embeddings
- store and search in a vector database
- retrieve relevant context from my own data
- chat through text and voice
- support multiple LLMs and embedding providers
- support multiple storage backends
- support evaluation, tracing, and feedback
- later support fine-tuning and advanced retrieval methods

Suggested Stack:
- Backend: Python + FastAPI
- Database: PostgreSQL
- Vector DB: Qdrant or pgvector
- Cache/queue: Redis
- OCR: Tesseract, PaddleOCR, or Surya
- PDF parsing: PyMuPDF, Unstructured
- LLM runtime: Ollama for local models
- Embeddings: BGE, E5, Nomic
- Reranker: BGE reranker
- Voice: Faster Whisper for STT, Coqui TTS for TTS
- Evaluation: RAGAS
- Tracing: Langfuse
- Agent/workflow orchestration: LangGraph
