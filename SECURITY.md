# Security Policy

RAG Builder is a technical preview. Security fixes are developed against the
latest default branch; older commits, forks, and private deployments do not have
a guaranteed support window or response SLA.

## Reporting a vulnerability

Please report suspected vulnerabilities privately through the repository
host's private security-advisory feature. If that feature is unavailable,
contact the project owner through a private channel listed on their repository
profile. Do not include credentials, customer data, exploit payloads, or other
secrets in a public issue.

Include the affected revision, deployment profile, impact, reproduction steps,
and any safe remediation suggestion. The maintainer will acknowledge the report
when practical, assess it, and coordinate disclosure after a fix is available.

For ordinary bugs without a security impact, use the public issue tracker and a
minimal sanitized reproduction.

## Deployment responsibility

Operators are responsible for secret management, TLS, network restrictions,
provider data-handling terms, backups, access logs, and applying upgrades. The
sample configuration is not a substitute for an environment-specific security
review.
